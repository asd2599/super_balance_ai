# [프롬프트 기반 AI 시트 탭 자동 생성기]

사용자가 자연어 채팅창(Prompt)을 통해 원하는 형태의 표(예: "이번 주 헬스장 운동 루틴 표 만들어줘")를 입력하면, AI가 최적화된 시트명, 헤더(열 구조), 초기 샘플 데이터(행)를 설계하여 Google Sheets에 새로운 탭(Sheet)으로 즉시 생성하는 기능입니다.

## User Review Required

> [!WARNING]
> - 신규 시트(탭)를 구글 스프레드시트에 새로 생성하는 API(`addSheet`) 권한을 사용하게 됩니다.
> - 사용자의 프롬프트를 받아 AI가 제목, 테이블 헤더 리스트, 초기 샘플 데이터 3줄을 구조화된 JSON 데이터로 디자인한 후 구글 시트에 복사해 넣는 프로세스로 기획했습니다. 괜찮으신가요?
> - 프론트엔드의 화면 좌측 사이드바 혹은 우상단에 전용 버튼(`+ AI 시트 자동 생성`)과 자체 커스텀 프롬프트 대화창(Prompt 창)을 띄우려고 합니다. 현재는 브라우저 기본 알림창인 `prompt()` 함수를 사용하여 심플하게 갈 계획입니다만 동의하시나요?

## Proposed Changes

### 백엔드 (FastAPI) 구조 변경

#### [MODIFY] backend/main.py
- **[NEW] Endpoint**: `POST /api/sheets/generate`
  - Request Body: `{"prompt": "사용자가 원하는 시트에 대한 자유로운 텍스트 (예: 스타트업 직원 관리 표)"}`
  - **AI 연동**: OpenAI를 호출하여 프롬프트에 맞는 `sheet_title`(시트 탭 명칭), `headers`([]), `initial_data`([[], [], []])를 응답해달라고 지시합니다. (JSON 형태로 강제)
  - **Google Sheets 연동**:
    1. `batchUpdate`의 `addSheet` 명령어를 사용해 새로운 탭을 1개 생성합니다.
    2. 생성된 시트에 AI가 반환한 헤더 및 초기 데이터를 2차원 배열(`values.update` 또는 `values.append`)로 밀어넣습니다.
    3. 완료 시 `{"message": "생성 완료", "sheetId": 12345}` 반환.

---

### 프론트엔드 (React) 구조 변경

#### [MODIFY] frontend/src/App.jsx
- **신규 UI 요소**:
  - 좌측 사이드바 시트 목록 상단 영역에 `✨ AI 시트 자동 생성` 가시적인 버튼을 추가.
- **모달/프롬프트(Prompt) 로직 구현**:
  - 버튼 클릭 시 기본 `window.prompt("어떤 형식의 표 시트를 새로 만들까요?")` 창을 띄워 사용자 입력을 받습니다.
  - 빈 문자열이거나 취소면 무시하고, 문장이 들어오면 API에 `{"prompt": "..."}` 정보를 실어 전송합니다.
- **로직 (Mutation)**:
  - 생성이 완료(`POST /api/sheets/generate`)되면, 즉시 전역 `fetchSheetList`를 호출하여 사이드바를 다시 그립니다.
  - 새로 만들어진 시트를 클릭하면 해당 탭의 초기 데이터들을 바로 볼 수 있도록 연결합니다.
- **대기 상태(Loading) 관리**:
  - 시트 생성까지는 AI의 분석에 따라 15초가량 소요될 수 있으므로, 해당 기간 동안 화면 우측 상단에 노란색의 로딩 알림이 노출되도록 기존 `loadingMutation` 로직을 공유합니다.

## Open Questions

- 새 탭을 생성할 때, 탭의 이름(`sheet_title`)이 반드시 고유해야 합니다. AI가 생성한 제목이 우연히 겹칠 경우 오류가 발생할 수 있는데, 필요할 경우 이름 뒤에 `_난수` 혹은 시간표기를 붙여서 중복을 방지하는게 좋을까요?
