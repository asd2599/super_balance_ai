# 🎮 게임 데이터 관리 및 AI 밸런싱 도구 과제면접 기획안

---

## ① 문제 정의

- **해결하려는 문제**: 
  수천 개가 넘어가는 게임 내 아이템, 몬스터, 경제 시스템 등 방대한 데이터의 '수치 밸런싱 작업' 중 발생하는 휴먼 에러 도출 및 초기 신규 콘텐츠 데이터 확장의 막막함 해결.
- **대상 사용자**: 
  게임 기획자 (시스템 기획자, 밸런스 기획자, 콘텐츠 기획자)
- **현재 업무 방식**: 
  엑셀이나 구글 스프레드시트에 수동으로 데이터를 기입하고, 함수(VLOOKUP 등)를 엮어 상호참조표를 만듭니다. 이후 직접 인게임에 넣어 시뮬레이션 돌려보거나 차트를 그려가며 감으로 밸런스를 잡습니다.
- **주요 불편 사항**: 
  수치가 하나만 잘못 꼬여도(가성비 폭망 무기, 골드 무한 복사 등) 전체 인게임 경제가 붕괴됩니다. 버전 변경 내역 추적이 어렵고(Git 도입 한계), 신규 스테이지 기획 시 초기 일과성 더미 데이터를 채우는 데 시간이 너무 오래 걸려 창의적 인력 낭비가 심합니다.

---

## ② 해결 방법

- **기획하는 AI 제품 또는 기능 설명**: 
  **"Super-Balance AI"** (스프레드시트 연동형 게임 밸런스 AI 매니저 시스템). 사람이 하던 기계적인 검수와 초안 작성을 AI가 돕는 Co-pilot 목적의 B2B 사내툴.
- **핵심 기능**:
  1. **AI 밸런스 이상 탐지 (Audit)**: 현재 시트의 전체 행들을 교차 스캔하여 수치 모순(비싼데 약한 무기 등)을 자동 경고.
  2. **AI 데이터 일괄 생성/수정**: "전체 체력을 20% 상향해줘" 같은 자연어 프롬프트를 정확히 연산된 2차원 배열로 변환.
  3. **데이터 이력 조회 (DB 로그)**: 누가, 언제, 어떤 의도로 수치를 수정했는지 Undo/Redo와 함께 영구 기록.
  4. **원클릭 인게임 리소스 추출**: 현재 상태를 즉시 무결성 CSV, JSON 포맷으로 클라이언트에 다운로드.
- **사용자 이용 흐름 (User Flow)**: 
  구글 시트 URL 연결 → 'AI 시트 생성' 버튼으로 원하는 형식의 테이블 뼈대 자동 구축 → 'AI 이상 탐지' 버튼으로 의도치 않은 가성비 붕괴 요소 스캔 및 화면 중앙 모달 리포트 확인 → 이상점 수정 후, 상단의 '⬇️ JSON/CSV' 버튼으로 인게임 엔진에 바로 이식.

---

## ③ 사용할 AI 도구

- **활용할 수 있는 AI 모델 또는 도구 설명**:
  - **OpenAI GPT-3.5-Turbo (또는 GPT-4o) API**: 고도화된 컨텍스트 이해 능력을 기반으로, 수학적 배열(오브젝트)을 읽고 룰 기반 이상점을 찾아내는 두뇌 역할로 채택.
  - **프롬프트 엔지니어링 전략**: AI에게 명확하게 "너는 최고 수준의 수치 밸런스 디자이너야"라는 페르소나를 부여하고, Temperature(창의성 지수)를 낮춰 기발한 답변 보다는 일관되고 엄격한 수치 로직 검증에 집중하도록 시스템 프롬프트 셋팅.

---

## ④ 프로토타이핑 계획

- **프로토타입 목적**: 
  단순한 챗봇이 아닌, AI가 실제로 엑셀 시트(2차원 배열) 형태의 수치적 함수 관계를 이해하고 JSON 스키마를 무결성 있게 리턴할 수 있는지(Anomaly Detection PoC) 구조적 검증.
- **구현 범위**: 
  Google Sheets API 연동 → React 기반 시각화 대시보드 표출 → FastAPI 기반 백엔드 단일 액션 분석 및 응답 모델링.
- **사용할 도구**: 
  Frontend(React, Vite), Backend(Python FastAPI, SQLite DB), External(Google Sheets API 연동 토큰).
- **입력 데이터 및 출력 방식**:
  - `입력`: 현재 편집 중인 구글 시트의 최신 2차원 배열 데이터(JSON 직렬화 형태로 백엔드 전달).
  - `출력`: 밸런스 오류가 감지된 행 번호(`row_index`)와 사유(`issue`)가 명확하게 명시된 순수 JSON 객체 (UI 모달로 텍스트 표출).

---

## ⑤ 프로토타이핑 방법

- **실제 프로토타입을 제작한 방식**: 
  프론트엔드와 백엔드를 완전 분리하여 로컬 호스트에서 구동 (웹 애플리케이션 형태). 
  백엔드는 Google Service Account 인증용 JSON을 서버 권한으로 물려, 브라우저가 직접 접근하지 못하게 보안을 챙기면서 실시간 시트 데이터를 가져오도록 설계했습니다. 가져온 데이터를 백엔드가 OpenAI API로 전송하고, 그 스캔 결과(JSON)를 다시 React가 넘겨받아 '⚠️ AI 밸런스 검사 리포트' 모달 팝업으로 사용자에게 띄워주는 방식으로 프로토타이프의 User Journey를 완벽히 구현했습니다.

## ⑥ 바이브 코딩 지시 작성 (예시 프롬프트)

> **※ 아래 텍스트 상자를 복사하여 AI에게 전달하면 동일한 프로토타입 제작을 돕습니다.**

```text
너는 지금부터 시니어 게임 서버/클라이언트 풀스택 엔지니어이자 전문 프롬프트 엔지니어 역할이야. 나는 게임 기획자와 밸런스 디자이너들을 돕기 위한 사내 데이터 테이블 관리툴 웹 어드민을 기획하고 있어. 아래 요구사항에 맞춰 프로토타입 전체 코드를 짜줘.

1. 기능 및 기술 스택:
- Backend: Python FastAPI와 SQLite(작업 히스토리 로깅용)를 사용해. Google Sheets API 토큰을 연동해서 시트 데이터를 읽고 쓰는 로직을 구현해.
- Frontend: React와 Vite로 만들어줘. 
- Core AI 연동: FastAPI 쪽에 OpenAI API를 연동하고, 시트의 배열 데이터를 통째로 프롬프트에 얹어서 "수치적 밸런스 이상점(Anomaly)"을 JSON으로 반환받는 핵심 로직(audit 함수)을 짜줘.

2. UI 요구사항:
- 좌측엔 연결된 시트 목록 네비게이션을 배치해.
- 메인 우측 영역 중앙에는 가져온 데이터 통계(표)를 보여줘.
- 상단 도구 막대(Top Bar) 좌측엔 "⬇️ CSV", "⬇️ JSON" 추출 버튼과 빨간색의 강력한 "⚠️ AI 밸런스 검사" 버튼을 한 줄로 나열해 (whiteSpace: nowrap 적용 필수).
- 우측 상단엔 DB와 연동된 "📜 작업 로그 조회", "↩️", "↪️" 버튼을 배치해.

3. 제약 조건 (프롬프트 인젝션 방지):
- AI에게 데이터를 전송할 땐 다른 부가적인 마크다운 말끝(``` 등)을 철저히 배제하고 무조건 JSON 객체(행 번호, 발생 이슈 딕셔너리)로 파싱되게끔 Temperature를 낮추고 지시문을 엄격하게 작성해줘. 발생 에러는 FastAPI에서 Catch해서 프론트엔드로 500이 아닌 안정적인 Error JSON을 리턴해야 해.
```
